个人博客攻略 - 无需编程知识，使用 Hexo 和 Github 搭建个人博客

本地准备

环境需要的软件安装

首先我们需要安装一些软件在电脑： Node.js npm git

Node.js 就是运行在服务端的 JavaScript
NPM Node.js的包管理工具，同NodeJS一起安装了
Git 是开源的分布式版本控制系统，用作项目版本管理

去 Node.js 和 Git 官网分别下载安装包安装即可

安装完成后，打开 cmd （命令提示符） ，分别输入命令检查是否安装成功
node -v npm -v git -v

输入命令：
npm install hexo-cli -g 使用 npm 安装 Hexo （这里-g 是全局安装）
hexo -v 检查 Hexo 安装是否成功
npm install hexo-deployer-git –save 安装hexo-deployer-git 自动部署发布工具

本地生成博客

创建一个文件夹作为博客文件夹，用于放置项目文件，右键该文件夹的空白处进入 Git Bash Here
输入命令：
hexo init 初始化 Hexo ，服务器在国外，可能会失败，如果失败了需要多试几次或使用其他手段

hexo s 打开本地服务器
根据提示信息可以通过在浏览器中打开http://localhost:4000/，可以看到欢迎页面，根据提示按 Ctrl + c 可以关闭
将本地的博客发布到Github
上面已经在本地搭建好了一个可以运行的博客，现在我们把它发布到Github上
在 Github 上搭建仓库
新建一个仓库 repository
仓库名字一定是需要自己的用户名加上 .github.io 即 name.github.io ，否则无法成功

Github 与本地绑定

在任意一个文件夹中右键空白处进入 Git Bash Here
输入ssh检查是否安装了SSH，默认在Git安装时已安装
生成 SSH ： ssh-keygen -t rsa -C “电子邮箱地址” 电子邮箱地址 是注册时的电子邮箱地址
接下来的提示信息让你输入什么的，不用输入直接按回车直到成功就可
上一步过后，在Windows用户文件夹下生成.ssh目录和其内部的文件，打开文件id_rsa.pub复制内容（SSH key）
打开 Github -> Setting -> SSH keys -> New SSH key 在 key 中输入上一步复制的内容（SSH key）， Titlle 任意起，Add添加
在Git命令行中，输入命令测试是否绑定成功：ssh -T git@github.com 如有提问要输入回答，yes即可，成功了会有 successfully 字样（同样因为网络等原因可能需要多式几次）

配置 _config.yml 文件

上面创建的本地项目文件里有一个 _config.yml 文件，用文本编辑器打开，编辑最后面的部分，改成：

github地址 就是github项目的网址加上.git github项目页面会有提示的
注意冒号后面是有一个空格的，这是yaml的语法要求
文件修改完成了，可以将其关闭了

在博客文件夹空白处右击鼠标进入 Git Bash Here
输入命令：
hexo g 生成文件

git config –global user.email “电子邮箱地址” 配置email
git config –global user.name “rchangl” 配置用户名

设置令牌：
进入页面 ithub -> Setttings -> Developer settings -> Personal access tokens -> Generate new token
Note 随便起一个名字，Expiration 有效期自行看着选，Select scopes 令牌权限全选（有些用不到但是全选也没事），点击生成令牌
成功后跳转的页面有一串字符，一定将他记下，只能看到这一次，再次打开页面后无法再看到

Git 命令行输入命令：
hexo d 本地文件上传到Github上面，会弹出一个账号密码页面，可以跳转到浏览器登录（同样因为网络问题可能需要多式几次）密码是上面设置的令牌密码
（如果这里 报错ERROR Deployer not found: git ，这是因为没安装hexo-deployer-git插件）
完成后github上的仓库会有已上传好的文件
完成部署
这时候已经可以用创建好的网址 即 用户名.github.io 访问博客内容了，但是现在还无法发布自己写的文章

在博客当中添加内容

在博客当中添加内容，思路是在本地添加好内容，再用 Git 同步至 Github 仓库上

本地博客中添加内容

在我们的博客文件夹中有很多文件和文件夹，现在我们只关注一个文件夹 source 里面存放的就是我们的个人文件，所有的个人文件都在里面。source文件夹下面还有一个 _posts 文件夹，这个文件夹里存放的就是我们的博客文件，里面有一个默认生成的 hello-world.md 文件，他会被生成一个我们之前看到的默认欢迎页面。
要添加我们自己的博客文件，只需要写好 Markdown 文件就可以了。

Markdown 文件的编写非常的简单，知乎、简书等很多平台都支持直接使用 Markdown 编辑，在搜索引擎中搜索 Markdown 教程，对着敲没几分钟就能学会；在Blibili上搜 Markdown ，视频教程也不过几分钟而已，这里不再讲解了。

将写好的md文件生成博客并发表到Github

像上面一样，进入 Git 命令行，然后输入命令：

hexo cl 用于清除缓存文件（db.json）和已生成的静态文件（public）
hexo g 生成文件
hexo d 同步至Github

网站实例：https://rchangl.github.io/
